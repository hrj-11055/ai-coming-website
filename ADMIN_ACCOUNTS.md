# 管理员账户说明

## 🔐 预设管理员账户

### 1. 超级管理员
- **用户名**: `admin`
- **密码**: `admin123456`
- **权限**: 所有功能
- **说明**: 拥有系统最高权限，可以管理所有内容

### 2. 内容编辑员
- **用户名**: `editor`
- **密码**: `editor123456`
- **权限**: 关键词管理、新闻管理
- **说明**: 专门负责内容编辑，无法修改系统设置

### 3. 系统管理员
- **用户名**: `manager`
- **密码**: `manager123456`
- **权限**: 关键词管理、新闻管理、系统设置
- **说明**: 拥有大部分管理权限，可以配置系统参数

## 🛡️ 安全建议

### 生产环境部署前
1. **修改默认密码**: 将所有账户的密码改为复杂密码
2. **删除测试账户**: 如果不需要，可以删除 `editor` 和 `manager` 账户
3. **服务器端验证**: 将账户验证迁移到服务器端
4. **HTTPS部署**: 确保登录过程使用HTTPS加密

### 密码修改方法
1. 编辑 `admin-accounts.js` 文件
2. 修改对应账户的 `password` 字段
3. 保存文件即可生效

## 📝 账户管理

### 添加新管理员
在 `admin-accounts.js` 中添加新账户：
```javascript
'newadmin': {
    password: 'newpassword123',
    role: 'manager',
    permissions: ['keywords', 'news'],
    name: '新管理员',
    lastLogin: null
}
```

### 修改权限
修改 `permissions` 数组：
- `['all']`: 所有权限
- `['keywords']`: 只能管理关键词
- `['news']`: 只能管理新闻
- `['settings']`: 只能修改系统设置

## ⚠️ 重要提醒

1. **不要开放注册**: 管理员账户只能通过代码添加
2. **定期更换密码**: 建议每3个月更换一次密码
3. **限制访问**: 确保只有授权人员知道管理后台地址
4. **监控登录**: 定期检查登录日志，发现异常及时处理

## 🔧 技术说明

- 账户信息存储在 `admin-accounts.js` 文件中
- 登录状态通过 `localStorage` 保存
- Token验证机制确保会话安全
- 支持"记住我"功能，方便日常使用
