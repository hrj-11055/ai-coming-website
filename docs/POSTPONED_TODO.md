## 后期改进（非紧急）

### 1. MySQL 存储迁移

**触发条件**：
- 日活超过 5000
- 并发写入导致 JSON 数据损坏
- 需要更复杂的数据查询

**好处**：
- 数据安全（事务支持）
- 并发处理能力
- 查询性能

**工作量**：
- 初始化 MySQL：1小时
- 数据迁移：30分钟
- 配置切换：30分钟

**操作步骤**：
```bash
# 1. 初始化数据库
npm run db:init

# 2. 迁移数据
npm run db:migrate

# 3. 切换到 MySQL 模式
pm2 stop ai-news-system
pm2 start server-mysql.js --name ai-news-system
```
