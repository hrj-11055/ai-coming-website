# AIcoming 产品文档

版本：`0.2.0`  
更新日期：`2026-03-20`

## 1. 产品定位

AIcoming 是一个围绕“信息获取、工具发现、内容消费、后台运营”展开的 AI 门户产品。

它当前不是单一聊天产品，也不是单一 CMS，而是由以下四个产品层组成：

1. AI 搜索与提示词辅助首页
2. AI 资讯与热点关键词页面
3. AI 工具集目录页面
4. 面向内容与运维的后台 API 能力

## 2. 目标用户

### 2.1 信息消费者

- 希望快速浏览 AI 资讯
- 关注每日热点关键词
- 希望直接收听每日播客摘要

### 2.2 工具发现用户

- 想按场景查找 AI 工具
- 需要区分国内/国外工具生态
- 希望快速直达官网

### 2.3 内部运营与维护人员

- 负责导入日报、维护数据、检查线上状态
- 负责排查播客链路与环境配置
- 需要通过后台接口和脚本完成运维动作

## 3. 当前产品结构

### 3.1 首页 `/index.html`

定位：

- 作为品牌首页和 AI 搜索入口
- 提供提示词辅助和引导式输入体验

当前价值：

- 降低用户开始使用 AI 搜索的门槛
- 为后续资讯、工具页提供统一导航入口

### 3.2 AI 资讯页 `/news.html`

定位：

- AIcoming 的核心内容消费页面

当前包含：

- 每日 AI 资讯列表
- 热点关键词可视化
- 今日播客状态与播放能力

关键体验：

- 用户可以查看当天资讯
- 用户可以在页面内直接触发或收听播客

### 3.3 AI 工具集 `/tools.html`

定位：

- 面向工具发现与分类浏览的目录页

当前组织方式：

- 按分类展示
- 每个分类同时区分国内区与国外区
- 点击工具卡片直达官网

当前实现边界：

- 展示数据目前来自前端静态目录 `frontend/modules/tools-catalog.js`
- 后端 `/api/tools*` 仍存在，但当前工具页渲染并不直接依赖该接口

这是一处必须明确记录的产品实现事实，也是本轮文档治理的重点之一。

### 3.4 关于页 `/about.html`

定位：

- 对外说明公司定位、合作方式与联系渠道

### 3.5 管理与运维入口

当前页面：

- `/admin-login.html`
- `/admin-analytics.html`
- `/admin-ipban.html`

当前价值：

- 提供运营登录、流量与地域统计、安全治理入口

## 4. 核心业务流程

### 4.1 日报内容流

1. 外部日报或数据源生成 JSON
2. 服务端导入或同步到当前运行数据
3. 新闻页对外展示
4. 后续可触发播客生成

### 4.2 工具目录流

1. 运营确定工具归类与国内/国外分区
2. 修改 `frontend/modules/tools-catalog.js`
3. `frontend/tools-page.js` 渲染页面
4. 用户在 `tools.html` 浏览并跳转

### 4.3 每日播客流

1. 读取服务器固定输入目录中的日报 JSON
2. DeepSeek 生成口播稿
3. MiniMax 异步生成音频
4. 服务器下载并处理音频
5. 上传 OSS 或回退到本地音频
6. 新闻页通过 `audio_url` 播放

### 4.4 微信公众号草稿流

1. 只读取 `Asia/Shanghai` 当天的日报 JSON 与当天播客 metadata
2. 将当天日报 JSON 自动整理成适合公众号的 Markdown 中间稿
3. 将当天播客 metadata 自动整理成播客版文章中间稿
4. 通过微信公众号 API 上传到草稿箱
5. 整个流程不依赖浏览器登录态，也不回退使用旧日期内容

## 5. 当前技术与运营边界

### 5.1 已稳定边界

- JSON 文件存储是当前唯一稳定主线
- `server-json.js` + `server/runtime.js` 是当前正式运行时
- 播客链路采用服务器优先治理
- 微信公众号草稿自动上传采用独立定时任务治理

### 5.2 非当前发布主线

- MySQL 能力仍保留，但不是当前版本交付重点
- 工具页暂未切换到后端动态数据源

### 5.3 文档边界

- 活跃文档只记录当前线上/主线可验证状态
- 旧方案与历史设计统一归档，不再作为实现依据

## 6. 当前产品风险与注意事项

### 6.1 工具页存在“双数据源概念”

- 展示用的是静态目录
- 后台仍保留工具 API

如果不在文档中说清楚，容易导致维护者误以为修改 `data/tools.json` 会直接影响工具页展示。

### 6.2 播客链路对服务器环境高度敏感

- 本地测试通过不代表线上生效
- `.env`、服务文件与线上接口状态必须一起看

### 6.3 运行时数据不应默认推入代码发布版本

- `data/`
- 播客元数据
- 音频文件
- 锁文件
- 备份文件

这些内容属于运行资产，不等同于代码资产。

## 7. 0.2.0 版本定义

`0.2.0` 是当前产品文档与版本治理收敛后的发布版本，重点不在“新增一个大型新模块”，而在于：

1. 统一三方代码状态
2. 修正 AI 工具集分区信息
3. 固化播客链路与相关脚本
4. 收敛活跃文档入口，降低文档漂移
